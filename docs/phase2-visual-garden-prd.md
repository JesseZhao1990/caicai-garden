# 菜菜菜园管家二期 PRD：菜地可视化美观展示

版本：v0.1  
日期：2026-07-01  
范围：Android 端二期，重点升级「菜园」页的可视化展示和地块浏览体验。

## 1. 背景与现状

一期已经完成家庭菜园管理 MVP：本地数据存储、天气获取、地块管理、种植批次、作物档案、智能提醒、7 天游历和操作记录。当前主体验以信息卡片为主，「菜园」页已经有一个基础 Canvas 菜地地图，能按地块展示作物类别、进度和少量作物形态，但整体仍偏工具型：

- 菜地只展示当前选中的单个地块，缺少完整花园/菜园空间感。
- 地块、作物、道路、装饰物之间没有整体布局关系，用户难以获得“这是我的菜园”的拥有感。
- 作物视觉表达已经有雏形，但还不够精致，缺少阶段差异、天气状态、任务状态和收获感。
- 当前交互以筛选地块和查看卡片为主，没有点击地块、展开详情、布置菜地等更自然的空间交互。
- 视觉风格仍是 Material 卡片风格，尚未形成接近模拟经营游戏的明亮、饱满、治愈感。

二期不优先扩账号、后端同步或 AI 图片识别，而是先把已有菜园数据转成更好看的“可视菜地场景”。

## 2. 风格参考

参考对象：

- 梦想城镇 / Township：参考其明亮饱和的模拟经营氛围、斜俯视场景、农场与城镇元素共存、作物收获感和装饰感。参考链接：[Playrix Township](https://playrix.com/games/township)、[Google Play Township](https://play.google.com/store/apps/details?hl=en_US&id=com.playrix.township)
- 我的花园世界：参考其花园经营、花卉种植、主题装饰、庭园布置和治愈向视觉表达。参考链接：[Google Play 我的花园世界](https://play.google.com/store/apps/details?hl=zh&id=com.jxhy.official)、[App Store 我的花园世界](https://apps.apple.com/cn/app/%E6%88%91%E7%9A%84%E8%8A%B1%E5%9B%AD%E4%B8%96%E7%95%8C/id6746321359)

设计原则：

- 借鉴氛围，不复制素材：采用原创菜地元素、原创作物图形和原创装饰，不复刻参考游戏的具体 UI、建筑、角色或图标。
- 斜俯视优先：用 2.5D / isometric 语言表达菜畦、阳台箱、盆栽和大棚，让用户第一眼看到空间布局。
- 真实种植数据驱动：所有视觉状态来自现有地块、批次、作物、生长阶段、天气和任务数据，避免纯装饰覆盖核心信息。
- 轻游戏化，不变成游戏：二期只做美观、布置、状态感和反馈，不引入金币、体力、复杂任务链和社交。

## 3. 产品目标

### 3.1 核心目标

让用户打开「菜园」页时，能看到一个明亮、好看、可交互的个人菜地总览，并能从场景中直接理解：

- 我有几个地块，它们在哪里。
- 每个地块正在种什么，长到什么阶段。
- 哪些地块今天需要关注。
- 哪些地块快要采摘或适合拍照记录。
- 我的菜园整体被装扮成什么样。

### 3.2 成功标准

体验标准：

- 用户无需阅读列表，也能从菜地图看出至少 80% 的地块占用状态。
- 用户点击任意地块后，能在 1 步内看到作物、生长阶段、进度、今日任务和快捷记录入口。
- 总览页首屏有明显的“我的菜园”空间感，而不是普通数据面板。
- 空菜地、成长中、需关注、可采摘四类状态视觉差异明确。

工程标准：

- 仍使用 Kotlin + Jetpack Compose 实现，不引入大型游戏引擎。
- 地图渲染在普通中端 Android 设备上保持流畅，首屏渲染目标小于 500ms。
- 可视化层可从现有 `GardenDataState` 派生，二期不强制做数据迁移；新增布局数据需要有默认生成策略。
- 所有可视元素在无网络、天气失败、本地示例数据下均可展示。

## 4. 用户与场景

目标用户：

- 家庭菜园用户：有露地、小院、屋顶或阳台种植需求。
- 阳台种植用户：地块少，但希望看到漂亮的箱栽/盆栽呈现。
- 新手种植用户：更依赖视觉化提示理解今天该干什么。

核心场景：

- 早上打开 App 看今日菜地状态。
- 到菜地前快速确认哪个地块要浇水、施肥或巡查。
- 给一块地拍照或记录操作。
- 新增种植后，希望在地图上看到作物出现。
- 希望把菜园布置得更好看，获得持续打开的动力。

## 5. 二期范围

### 5.1 Must Have

1. 菜园总览地图

- 在「菜园」页首屏展示完整菜园场景，而不是只展示单个选中地块。
- 支持露地、盆栽、大棚、阳台箱四类地块外观。
- 默认根据现有地块生成稳定布局：2-6 个地块时自动排列；超过 6 个时支持横向/纵向滚动。
- 地块上直接展示作物视觉：叶菜、瓜豆、茄果、根菜、葱蒜/香辛类至少 5 类作物图形。
- 地块空置时展示翻土/播种点状态。

2. 生长阶段视觉

- 根据现有 `PlantingInsight.progressPercent` 和 `GrowthStage` 输出作物大小、密度、颜色和果实状态。
- 至少支持 4 个视觉阶段：新种下、幼苗/缓苗、生长期、采收期。
- 对连续采收作物提供更强的成熟提示，例如果实点缀、采摘光标或成熟标签。

3. 地块状态提示

- 今日高优先级任务的地块显示醒目的状态角标。
- 浇水类任务显示土壤偏干或水滴提示；天气巡查类任务显示天气/防护提示；采摘类任务显示成熟提示。
- 角标不遮挡作物主体，不影响地块点击。

4. 地块详情底部面板

- 点击地图中的地块，打开地块详情底部面板。
- 面板内容包括：地块名、种植方式、土壤、面积、当前批次、作物名、品种、生长第几天、阶段、进度、预计采摘窗口、今日任务、快捷记录按钮。
- 支持从面板直接进入“添加种植”“记录操作”“结束批次”。

5. 视觉风格升级

- 菜园页采用游戏化场景背景：草地、泥土、道路、围栏、花坛、工具架、水桶、木牌等原创元素。
- 色彩从现有 Material 绿色扩展为多色明亮体系：草绿、泥土橙、天空蓝、花朵粉、木质棕、成熟果红。
- 保留底部导航和核心功能入口，避免用户迷失。

### 5.2 Should Have

1. 布置模式

- 菜园页增加“布置”入口。
- 支持移动地块位置，选择道路、围栏、木牌、水桶、花坛、灯串等装饰。
- 二期只做本机保存，不做商店、不做付费和社交展示。

2. 主题皮肤

- 预置至少 3 个主题：清晨菜园、盛夏菜园、暖棚小院。
- 主题影响背景、道路、围栏和装饰色彩，不改变数据。

3. 天气氛围

- 根据天气数据在地图上体现晴天、降雨、高温、低温等轻量氛围。
- 天气氛围必须可关闭或降低动效，以保证性能。

4. 地图缩放与拖动

- 手机小屏下支持轻量拖动查看大菜园。
- 二期缩放可选，优先保障点击地块和横纵滚动。

### 5.3 Could Have

- 作物图鉴入口：从作物视觉点击进入作物档案。
- 生长相册入口：在地块详情中展示最近一次拍照记录。
- 简单动效：作物轻微摇摆、完成任务时轻微闪光、切换主题时淡入。

### 5.4 Won't Have

- 不做账号、多设备同步。
- 不做多人拜访、交易、排行榜。
- 不做金币、经验、体力等完整游戏经济。
- 不做复杂 3D 或 Unity/游戏引擎重构。
- 不做真实图片识别和病虫害 AI 诊断。
- 不引入参考游戏的素材、角色、建筑或商业化系统。

## 6. 信息架构

二期后「菜园」页结构：

1. 顶部状态栏

- 菜园名称、地点、天气简报、今日任务数量。
- 入口：编辑菜园、布置模式。

2. 菜园总览场景

- 可滚动/可点击的 2.5D 菜地图。
- 地块、道路、装饰、天气氛围、状态角标。

3. 地块详情底部面板

- 默认收起，只露出选中地块摘要。
- 点击地块后展开。

4. 今日关注

- 地图下方保留精简任务列表，只展示最重要 3 条。
- 更多任务仍进入「今日」或「日历」页。

5. 操作入口

- 添加种植。
- 添加地块。
- 记录操作。
- 布置菜地。

## 7. 功能需求

### 7.1 菜园总览地图

用户故事：

- 作为用户，我希望打开菜园页就看到我的全部菜地，而不是一个抽象列表。
- 作为用户，我希望地块上的作物长得不一样，这样我能快速分辨番茄、黄瓜和叶菜。
- 作为用户，我希望空地看起来像可以马上种东西的土地。

验收标准：

- 有 0 个地块时，展示空菜园场景和“添加地块”入口。
- 有 1-6 个地块时，地图自动生成完整布局，并在首屏可看到主要区域。
- 每个地块都可点击，点击后选中态明确。
- 地块视觉由 `Plot.growingStyle` 决定基础外观。
- 作物视觉由 `CropProfile.category/name` 和 `PlantingInsight.progressPercent/stage` 决定。
- 同一地块有多个批次时，按最多 3 个作物条带或区域混种展示，超出部分在详情面板中列出。

### 7.2 作物阶段视觉

用户故事：

- 作为用户，我希望作物从播种到成熟在地图上逐步变化。
- 作为用户，我希望快能采摘的作物看起来更有收获感。

验收标准：

- `progressPercent < 15%`：播种点或小芽。
- `15%-45%`：幼苗，植株小、密度低。
- `45%-75%`：生长期，叶片饱满或藤架明显。
- `>=75%`：采收期，果实/根菜/叶丛更饱满，显示成熟提示。
- 高温或缺水任务存在时，土壤颜色和状态提示有差异。

### 7.3 地块详情底部面板

用户故事：

- 作为用户，我点击地块后，希望看到这块地最重要的信息和操作。

验收标准：

- 面板展开后不超过屏幕高度 65%，可滚动。
- 有作物时展示作物列表和每个作物的进度。
- 无作物时展示空地状态和添加种植入口。
- 今日任务按优先级排序，高优任务突出。
- 快捷操作至少包括：记录浇水/施肥/采摘/拍照、添加种植、结束批次。

### 7.4 布置模式

用户故事：

- 作为用户，我希望把地块摆得更像自己的菜园。

验收标准：

- 点击“布置”进入编辑状态，底部出现工具栏。
- 用户可以选择地块、道路、装饰、主题四类工具。
- 地块支持拖拽移动到网格位置。
- 装饰支持放置、移动、删除。
- 点击保存后本地持久化；取消后回到编辑前布局。
- 没有自定义布局时，仍使用自动布局生成。

## 8. 数据与技术方案建议

### 8.1 新增数据模型

建议新增可选数据，不破坏一期数据：

```kotlin
data class GardenVisualState(
    val gardenId: String,
    val themeId: String = "morning",
    val plotLayouts: List<PlotLayout> = emptyList(),
    val decorations: List<GardenDecoration> = emptyList()
)

data class PlotLayout(
    val plotId: String,
    val gridX: Int,
    val gridY: Int,
    val gridW: Int = 2,
    val gridH: Int = 2,
    val rotation: Int = 0
)

data class GardenDecoration(
    val id: String,
    val type: DecorationType,
    val gridX: Int,
    val gridY: Int,
    val rotation: Int = 0
)
```

持久化建议：

- 继续使用 SharedPreferences JSON，新增 `visualState` 字段。
- 读取旧数据时，如果没有 `visualState`，运行 `DefaultGardenLayout.generate(plots)` 得到稳定默认布局。
- 稳定布局依据 `plot.id` 排序，避免每次启动地块位置变化。

### 8.2 Compose 组件建议

建议拆分：

- `GardenSceneScreen`：二期菜园页容器。
- `GardenSceneCanvas`：总览地图 Canvas。
- `IsometricPlotRenderer`：地块绘制。
- `CropSpriteRenderer`：作物形态绘制。
- `GardenDecorationRenderer`：装饰物绘制。
- `PlotDetailSheet`：地块详情底部面板。
- `GardenEditToolbar`：布置模式工具栏。
- `GardenThemePalette`：主题色与场景 token。

### 8.3 性能要求

- Canvas 绘制中避免频繁分配复杂对象。
- 作物图形按类别复用绘制逻辑，二期不需要为每个作物单独 bitmap。
- 动效默认轻量，避免无限复杂动画。
- 地图尺寸超过屏幕时使用可滚动容器或手势偏移，避免一次绘制过大不可见区域。

## 9. 视觉设计要求

二期视觉目标升级为“梦幻而精致的 3D 菜地农场规划效果图”：整体采用高品质景观规划模型感，俯视等距视角，布局整齐但自然，风格可爱但高级。实现上不要求直接引入 3D 引擎，可先使用高质量 3D 主视觉资产 + Compose 信息层叠加的方式落地。

### 9.1 色彩

主色建议：

- 草地绿：`#6FBE5B`
- 深叶绿：`#2F7A44`
- 泥土棕：`#9C633B`
- 木质棕：`#8A5B35`
- 天空蓝：`#8FD7F5`
- 花朵粉：`#F58BB2`
- 成熟果红：`#E85A3E`
- 奶油白：`#FFF5DA`
- 阳光黄：`#F6C969`

### 9.2 形状与质感

- 主场景使用等距俯视构图，像精品农场规划模型，而不是普通平面卡片。
- 菜地包含多个分区菜畦、木质高架种植床、石板小路、自动灌溉管线、小型温室、木栅栏、农具棚和蔬菜标识牌。
- 作物需要茂盛、健康、清晰可辨，至少覆盖番茄、胡萝卜、生菜、卷心菜、南瓜、辣椒、茄子、黄瓜藤架、玉米、草莓和香草植物。
- 泥土需要有干净真实的颗粒/湿润质感，叶片和灌溉系统可体现水珠细节。
- 蔬菜标识牌使用图形或空白牌，避免生成或绘制错误文字。
- 避免杂乱布局、枯萎植物、低清晰度、模糊、过度写实脏乱农田、畸形蔬菜、暗色调、人物遮挡、水印和 logo。

### 9.3 信息层级

- 地图主体优先，卡片信息后置。
- 地块角标只展示状态，不堆文字。
- 文案尽量短，例如“需浇水”“可采摘”“雨后查”。
- 详情面板承载解释文字，地图只承载状态。

## 10. 里程碑

### M1：总览地图替换当前单地块地图

- 完整菜园场景。
- 多地块自动布局。
- 地块点击和选中态。
- 作物按类别/进度展示。

### M2：地块详情与状态提示

- 底部面板。
- 今日任务角标。
- 快捷记录入口。
- 空地/关注/采摘状态完善。

### M3：布置与主题

- 布置模式。
- 本机保存布局。
- 3 套主题。
- 轻量天气氛围。

## 11. 风险与取舍

- 风格过游戏化会稀释工具价值：二期不做经济系统，只做视觉和空间交互。
- Canvas 手绘维护成本较高：优先抽象类别级绘制，不为每个作物做独立复杂素材。
- 小屏信息拥挤：地图只放状态，详情放文字。
- 性能风险：默认少动效，先保证静态场景精致。
- IP 风险：只提炼“明亮、斜俯视、治愈经营”的风格，不使用参考游戏素材。
