# 实体菜园编辑器设计方案

日期：2026-07-02

## 目标

把菜园页从“功能按钮 + 小型布局板”改成“实体菜园主人的可编辑菜园场景”。画面参考 QQ 农场这类竖屏农场的清晰层次和亲和力，但不照搬其社交游戏系统，也不保留分享、音乐、相机、菜单、积分、宠物、商店等入口。

核心目标：

- 菜园本身成为首屏主体，撑满手机主要可视区域。
- 地块不拉伸、不变形，8x8 菜畦完整可见。
- 点击、拖拽、摆放、旋转、缩放都清晰易用。
- 作物随真实种植天数逐步长大，视觉上能看出幼苗、旺长期、结果期差异。
- 布局对应实体菜园，强调“真实菜园管理”，不是虚拟游戏农场。

## 视觉方向

### 画面结构

竖屏单页以菜园场景为中心：

1. 顶部轻量信息栏
   - 菜园名称。
   - 今日待办/成长状态摘要。
   - 编辑状态提示。

2. 中部大画布
   - 占屏幕主要高度。
   - 使用固定等距视角，菜地以菱形场景呈现。
   - 草地、灌木、树影作为边界氛围，但不遮挡 8x8 主地块。

3. 底部编辑工具栏
   - 横向工具托盘。
   - 只放实体菜园相关操作：查看、移动、种植床、温室、农具棚、标识牌、清空、重置。
   - 不出现游戏社交/经济入口。

### 美术标准

- 风格：温暖、清新、低多边形 3D / 2.5D 卡通写实。
- 配色：草地暖绿、土壤深褐、木头暖棕、成熟果实红/橙/紫作为点缀。
- 光照：左上方柔和阳光，物体有统一方向投影。
- 地块：每块土地区分顶面和侧面，形成厚度，不用平铺方块。
- 作物：不再用简单贴图覆盖，每种作物按阶段绘制不同高度、叶片密度和果实数量。

## 交互设计

### 默认查看模式

- 单指拖拽空白处：平移视图。
- 双指捏合：放大/缩小视图。
- 点击菜畦/设施：选中，并显示底部状态面板。
- 长按或直接拖拽菜畦/设施：移动到目标格。

### 编辑模式

选择工具后进入明确编辑状态：

- 移动：拖动任意已有组件换格。
- 种植床：点击空地放置当前作物种植床。
- 温室/农具棚/标识牌：点击空地放置。
- 旋转：选中组件后显示左转、右转、归正按钮。
- 清空：点击格子移除组件。

木栅栏、石板路、水管暂不作为可摆放组件，因为这些线性组件需要方向、连接、转角和自动拼接系统。没有这套系统时会破坏编辑体验。

### 命中与拖拽规则

- 只允许组件落在 8x8 主地块内。
- 拖拽时目标格高亮。
- 松手后若目标格已有组件，则互换或覆盖；当前版本采用覆盖移动，保持逻辑简单。
- 画布缩放和平移不改变地块比例。
- 组件旋转只影响绘制角度，不改变格子占位。

## 作物成长设计

作物成长使用现有 `PlantingInsight.progressPercent`：

- 0% - 20%：幼苗期，矮小嫩叶，土面可见。
- 21% - 50%：营养生长期，叶片变多，植株高度增加。
- 51% - 80%：开花/坐果期，出现支架、花朵、小果实。
- 81% - 100%：成熟/采收期，果实数量和颜色更明显。

不同作物需要有可识别差异：

- 番茄：支架 + 红果。
- 黄瓜：藤架 + 藤蔓。
- 生菜/卷心菜：低矮叶球。
- 胡萝卜：羽状叶 + 橙色根尖。
- 茄子/辣椒：深色或红色果实。
- 草莓：低矮叶片 + 红色小果。

## 当前实现策略

### 本次落地版本

在不引入完整游戏引擎的前提下，先把现有 Compose 场景重构为稳定 2.5D 编辑器：

- 删除游戏化入口和非实体菜园控件。
- 使用单一大画布替代小卡片画布。
- 固定等距投影，按宽度和高度共同计算格子尺寸，保证不变形。
- 用程序绘制高架床、土壤厚度、作物成长和阴影，避免直接贴一张带菜背景图。
- 工具栏改为实体菜园组件托盘。
- 自测拖拽、缩放、旋转、完整显示。

### 后续高质感版本

如果要继续逼近设计稿级 3D 效果，需要接入 SceneView/Filament：

- 使用 GLB 模型资产替换程序绘制。
- 固定 orthographic isometric camera。
- 用真实光照、阴影和深度排序解决“假 3D 变形”。
- 作物成长通过模型阶段或节点缩放实现。

本次实现会保留数据与交互边界，后续迁移到 SceneView 时不需要重写业务数据。

## 验收标准

- 首屏菜园画布明显大于旧版，主要视觉面积用于菜地。
- 8x8 地块完整显示，不能被裁切，不能被纵向拉伸。
- 菜和设施不是背景图的一部分，必须能点选、拖拽。
- 选中的菜畦可以旋转、归正。
- 双指缩放后仍能拖拽组件。
- 作物会根据种植进度改变大小、叶片和果实。
- 页面没有分享、音乐、相机、菜单、积分、宠物、商店等入口。
